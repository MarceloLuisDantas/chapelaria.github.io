Bem vindos a Chapelaria

Esta pagina é dedicada a servir como uma versão menos formal do meu currículo. Aqui eu falo um pouco sobre mim, minha trajetória e meus interesses. Aqui também esta salvo meus projetos e minhas demos de NCurses e Raylib.

Sobre Mim

Com experiência em programação desde 2014, iniciei meus estudos com Python, explorando desenvolvimento de interfaces gráficas (Kivy e Tkinter), aplicações web (Flask) e jogos (PyGame). Durante o ensino técnico no IFPB, tive contato com Pascal, Java e Python, onde desenvolvi preferência por linguagens de baixo nível e sintaxe simplificada. Minha trajetória inclui experimentações com Rust, Nim e outras linguagens modernas, culminando no domínio de C como linguagem principal para projetos pessoais, utilizando NCurse e Raylib. Na faculdade, aprofundei-me no desenvolvimento de sistemas, compiladores e ferramentas.

Minha abordagem valoriza a simplicidade e a eficiência, preferindo linguagens que oferecem liberdade e controle sobre o desempenho. Tenho especial interesse em projetos que exigem confiabilidade e qualidade, onde a eficiencia e performance é crucial. Minhas principais áreas de interesse incluem:
  • Compiladores: Desenvolvimento de ferramentas de tradução e interpretação de linguagens;
  • Sistemas Operacionais: Estudo e criação de sistemas eficientes para diferentes plataformas;
  • Jogos: Desenvolvimento de engines e mecanismos de baixo nível;
  • Emuladores: Implementação de software de emulação para sistemas legados.

C é a linguagem a qual possuo maior interesse, sendo a que utilizo para qualquer projeto pessoal que não seja uma ferramenta.

Python é a minha linguagem de preferência quando preciso criar ferramentas ou automatizar tarefas.

Golang é a minha linguagem de preferência para desenvolver sistemas que exigem confiabilidade e segurança.

Nim foi a linguagem a qual eu mais me dediquei e fiz projetos no período que utilizei. Dentre eles, meu primeiro projeto, o guia: Aprendendo Nim e o meu Interpretador

Elixir foi a minha primeira linguagem funcional. Foi com ela que aprendi os principais conceitos e a forma de pensar funcional

Rust é das alternativas a C, a que mais tenho contato e atualmente considero a melhor opção para desenvolvimento de sistemas.

Zig é a linguagem que mais estudei ao procurar alternativas a C e tenho cuiriosidade atualmente, no entanto ainda está em beta e acesso antecipado

Assembly: é a família de linguagens que mais tenho vontade de utilizar profissionalmente, especialmente no contexto de compiladores.

Meus Projetos

Sector Seven: Sector Seven é o meu primeiro projeto útil. Um sistema de build e testes para C, focado em ser simples de usar e aprender, não para substituir sistemas mais robustos como Make ou Ninja, mas para ajudar iniciantes que querem aprender C e não querem sofrer com a complexidade dos sistemas de build atuais.

Kokonoe Demo: Kokonoe foi minha primeira tentativa de criar um interpretador. Kokonoe é um interpretador de um subset de Assembly inspirado em MIPS.

GameOfLife: Um simples simulador de Conway's Game of Life. Uma demo que fiz para aprender o básico de como utilizar NCurses com C. É a recriação de um projeto antigo feito em Rust.

Dungeon: Um gerador de dungeons estilo Rogue. A ideia era criar um simples roguelike tradicional.

TTT: Um simples jogo da velha, uma demo para entender melhor como criar jogos com Raylib.

C-Codedump: Esse repositório é apenas um grande codedump de um monte de testes e coisas aleatórias que eu faço em C. Tem algumas estruturas de dados, testes de NCurses, experimentos e coisas do tipo.

Sector Seven (Old): Primeira versão do Sector Seven está arquivada. Esta versão é extremamente mal feita, fiz ela enquanto aprendia o que precisava sobre Python. Não estava escalando para o que eu precisava, porém não quis deletar, porque sim.

Facila: Minha primeira ferramenta CLI. Foi feito para ajudar na cadeira de Programação 2, onde utilizávamos uma ferramenta meio chata, então criei isso para ajudar a enviar os roteiros e fazer os testes.
